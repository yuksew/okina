# プロジェクト: 個人用 株式投資支援システム（設計フェーズ）

## 私の背景
- PHP/Laravel 出身。Clean Architecture 志向。API 設計・DevOps に手を動かす。
- 今回は Cloudflare + Hono（TypeScript）を軸に検討中。

## このプロジェクトの原則
- 検証優先: バックテスト/ペーパートレードで優位性を確認するまで実資金を入れない。
- 段階導入: 手動判断支援 → 半自動 → 自動執行。フェーズをスキップしない。
- 安全側: 自動執行にはキルスイッチ・発注上限・異常検知を必須とする。
- 設計判断は docs/ にファイルとして残す。対話ログに埋もれさせない。

## 進め方
- 前提（Cloudflare/Hono/単一ユーザー）は疑ってよい。より良い代案があれば理由付きで提案する。
- 現行仕様が不確実な箇所は web で裏を取ってから設計に反映する。取れなければ明記する。
