## 1. 戦略の基本方針

エッジ仮説を持たない前提のため、**「独自の予測」ではなく「規律の自動化」を優位性の源泉とする**。具体的には:

- 学術的・実務的に長期のエビデンスがあるファクター（モメンタム、トレンドフォロー）に基づく既知のルール型戦略を採用する。
- 判断は日次データ・月次（一部週次）実行に限定し、裁量ゼロで機械的に運用する。
- システムの価値は「シグナルの正確な計算」「ルール逸脱の検知」「実行の負担軽減」に置く。

**正直な注意事項**: これらの戦略は「過去に機能した」ものであり、将来の優位性は保証されない。特にタイミング系戦略は2010年代以降、バックテスト時代より成績が劣化しているという指摘が多い。本プロジェクトの合格ラインは「市場平均を大きく上回る」ではなく「**許容DD内（〜20%）でリスク調整後リターンがベンチマークと同等以上、かつ規律的に運用し続けられる**」に置く。

## 2. 戦略候補（バックテストで比較検証する4案）

| # | 戦略 | ルール概要 | 実行頻度 | 期待される性質 |
|---|---|---|---|---|
| S1 | **ベンチマーク: Buy & Hold + 年次リバランス** | 株式ETF + 債券/現金を固定比率（例 70/30）で保有、年1回リバランス | 年次 | 比較基準。全候補はこれに対しリスク調整後で優位である必要 |
| S2 | **トレンドフィルタ型（Faber式 GTAA簡易版）** | 各資産（米株/先進国株/債券/リート/金 等のETF 4〜6本）を「価格 > 10ヶ月移動平均なら保有、下なら現金」 | 月次（月末） | リターンは市場並み、**DDの大幅圧縮**が主目的。DD許容20%と整合的 |
| S3 | **デュアルモメンタム（GEM系）** | 米株/米国外株/債券ETFの12ヶ月リターンを比較し、絶対モメンタム（対安全資産）＋相対モメンタムで1本に集中 | 月次（月末） | 過去実績は良いが集中リスクと近年の劣化指摘あり。要検証 |
| S4 | **ETFモメンタムローテーション（Top-N）** | セクター/資産クラスETF 8〜12本から、3〜12ヶ月モメンタム上位2〜3本を保有 | 月次 | S3の分散版。回転率が上がるためコスト・税の影響を精査 |

- スイング志向（数日〜数週）への対応: 上記はいずれも月次だが、S2/S4は週次実行版も検証する（半自動運用の負荷上限は「週1回の発注アクション」とする）。
- 個別株は**フェーズ2以降の検討事項**とし、MVPではETFのみ。資金〜100万円で個別株分散は非効率、かつ無料データ・検証の複雑性が跳ね上がるため。
- **日本上場ETFもMVPから除外**。合法・無料で10年分の日次データを取る手段が存在しないため（裏取り済み、02-research §2）。ユニバースは**米国上場ETFのみ**とし、J-Quants課金（¥1,650/月〜）の価値はフェーズ2以降に再評価。

## 3. 口座配置の設計（NISA×ローテーションの注意点）

新NISA成長投資枠は**売却しても年間投資枠（240万円/年）は戻らない**（生涯枠は翌年復活）。月次ローテーション戦略をNISA内で回すと年間枠を急速に消費するため:

- **コア（NISA）**: S1的な長期保有部分（低回転）を配置。非課税メリットを最大化。
- **サテライト（特定口座・源泉徴収あり）**: S2〜S4のローテーション部分を配置。回転しても確定申告不要。
- コア:サテライト比率は検証結果を見て決定（初期仮説: 50:50〜70:30）。
- 裏取り結果の補足（02-research §1）: NISA×米国ETFのAPI自動発注の公認手段は存在しないため、コア・サテライトとも**発注は手動**で確定。ただしサテライト（特定口座）はmoomoo/Webullの公認APIで将来自動化できる余地があり、「税務優先 vs 自動化」のトレードオフは当初想定より緩い。

## 4. リスク管理ルール

| 項目 | ルール |
|---|---|
| 最大DD許容 | ポートフォリオ全体で **-20%**。-15%で警告通知、-20%到達で全ポジション見直し（戦略停止判断）を強制通知 |
| ポジション上限 | 1 ETFあたり全資産の40%まで（S3の集中型を除く。S3採用時は別途定義） |
| 現金退避 | S2/S3の絶対モメンタム/トレンドフィルタが機能した場合は現金（またはMMF/短期債ETF）へ |
| 発注上限 | 1回の発注は全資産の40%まで。超える発注案が出た場合はシステムが警告 |
| ルール逸脱検知 | 「シグナルと実際の保有の乖離」を毎日チェックし、乖離があれば通知（発注忘れ・裁量介入の検知） |
| 為替 | USD建て資産の円換算評価を日次で行う。為替ヘッジはしない（コスト対効果が小さい規模のため） |

## 5. 検証基準（バックテスト → ペーパートレード）

### 5.1 バックテスト
- **期間**: 最低15年（2008年金融危機を含めること）。Tiingoは米国ETFを30年超・調整済みで提供するため充足可能（裏取り済み）。ETFの設定日が足りない場合は先行する類似ETF・指数で補完し、補完区間を明記する
- **データ**: 配当・分割調整済み（adjusted close）。**Tiingo Free（adjClose、公式API）を主ソース、Stooqをクロスチェック用**とする（02-research §2）
- **コストモデル**: 売買手数料（国内証券の実レート）、スプレッド0.05〜0.1%、為替手数料を含める
- **税モデル**: 特定口座部分は譲渡益20.315%を反映したネットリターンも算出
- **過剰適合対策**: パラメータ（移動平均期間、モメンタム期間、Top-N数）は文献の標準値を使い、**最適化しない**。感応度分析（パラメータ±50%でも結論が変わらないか）のみ行う
- **評価指標**: CAGR / 最大DD / シャープレシオ / カルマーレシオ / 回転率 / 最長アンダーウォーター期間

### 5.2 合格ライン（バックテスト）
1. 最大DD ≤ 20%（コスト・税込み）
2. リスク調整後リターン（シャープ）が S1 ベンチマーク以上
3. パラメータ感応度分析で結論が反転しない
4. 回転率が「週1発注アクション以内」の運用負荷に収まる

### 5.3 ペーパートレード
- **期間**: 最低3ヶ月、推奨6ヶ月（月次戦略ならシグナル3〜6回分）
- **方法**: 本システムがシグナル→通知→仮想発注を記録。実際の発注はしない
- **合格ライン**:
  1. シグナル計算がバックテストロジックと完全一致（実装バグゼロ）
  2. 通知→（仮想）発注のオペレーションを1回も欠かさず実行できた（規律の実証）
  3. 仮想約定価格と翌日実勢の乖離（実行スリッページ）が想定コストモデル内
- ペーパー合格後、実資金は**まず30万円以下**で3ヶ月運用 → 問題なければ全額へ段階増額
