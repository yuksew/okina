## 決定内容: コアS5 + サテライトブレンド

| 区分 | 口座 | 中身 | 配分（初期値） | 実行頻度 |
|---|---|---|---|---|
| コア | NISA成長投資枠（SBI/楽天。NISAは両社とも米国ETF売買無料） | **S5: SPY 30% / AGG 70% 静的配分** | 資産の **75%** | 年1回リバランス＋乖離±5%超で臨時通知 |
| サテライト | **ウィブル証券**・特定口座（源泉徴収あり） | **ブレンド: S2 GTAA5(10ヶ月SMA) 50% + S4 Top4ローテーション 50%** | 資産の **25%** | 月末シグナル→通知→手動発注 |

- 初期資金100万円の場合: コア75万円 / サテライト25万円。
- 口座間の資金移動リバランスは原則行わない（NISA年間枠の消費を避ける）。年次レビューで配分乖離が大きければ新規入金側で調整。
- コア:サテライト=75:25は初期値であり、P3の実運用レビューで見直し可能。サテライト0%（S5単独）への縮退はいつでも可能な設計とする。

## 判定の要旨（詳細は08参照）

1. 厳密な合格ライン（DD≤20% かつ シャープ≥S1）を単独で満たしたのは **S5（NISA静的配分）のみ**（CAGR 5.8% / シャープ0.88 / DD 19.2%）。非課税・コストほぼゼロの構造優位は月次ローテーションでは埋まらなかった。
2. ただしS5の防御は「株安時に債券高」レジームに依存する（2022年型の株債同時安では効かない）。**ブレンド（トレンド退避型）は防御メカニズムが異なる**ため、25%のサテライト併用に分散価値がある（ブレンド50:50: 課税後CAGR 6.3% / DD 15.0%、moomooコスト前提）。
3. S3（GEM）は感応度含め全滅のため**正式除外**。
4. サテライトの執行先は当初moomoo特定口座（片道約0.14%）としたが、業務停止命令により**ウィブル証券に変更**（改訂履歴参照）。大手3社特定口座（約0.50%）では回転コストが年1%超になり不成立、という判断自体は不変。

## 改訂履歴

### 2026-07-12: サテライト執行先を moomoo → ウィブル証券へ変更

- 事由: 金融庁が2026-06-19、moomoo証券に一部業務停止命令。**新規口座開設の勧誘・受付を2026-09-18まで3ヶ月停止**（NISA対象外商品の虚偽勧誘、疑わしい取引の未検証、サイバーセキュリティ不備が理由。[日経](https://www.nikkei.com/article/DGXZQOUB18BI20Y6A610C2000000/) / [Bloomberg](https://www.bloomberg.com/jp/news/articles/2026-06-19/TGVBXWT9NJLS00)）。
- 代替の検証: ウィブル証券は受付継続中（2026-07-12確認）。採用ブレンドを**ウィブル現行手数料0.22%で再計算した結果、CAGR 6.1% / シャープ0.63 / DD 15.3% と moomoo比-0.2pt程度で合格圏内のまま**（`pnpm exec tsx src/cli/webull-check.ts` で再現可能）。さらに2026年7月下旬に米国株手数料無料化の報道あり（実現すればmoomooより低コスト化）。ウィブルもOpenAPI提供の将来自動化候補（02-research §1）。
- 留意点: (a) ウィブルの日本株APIは現物のみ・NISA非対応だが、サテライトは特定口座・米国ETFのみなので影響なし。(b) 処分理由を踏まえ、moomooは受付再開後も優先度を下げる。(c) Webull手数料無料化の公式確定は口座開設時に再確認（09-cost §4）。

## 誠実性の記録（将来の自分への注意）

- S5の「30/70」はバックテストと同じウィンドウで選んだ比率（in-sample選択）。静的配分の株式比率は連続的で感応度は素直だが、「19.2%がぴったり20%以内」に意味があるわけではない。**実運用でDDが20%を超えても想定内の誤差**として扱うこと。
- サテライト25%は「学習・分散・自動化パス維持」のための配分であり、期待リターン向上の根拠は弱い。ブレンドが3年連続でコアに劣後しても直ちに失敗ではないが、**P3の年次レビューで「規律通り運用できているか」だけは厳格に見る**。
- 評価ウィンドウ（2006〜2026）は米国株とドル建て債券に有利な期間だった。将来レジームが変われば数字は再現しない前提で、実資金投入はペーパートレード合格後・30万円以下から。

## P2で実装するシグナル・監視要件（この決定から確定するもの）

1. 月末: サテライトのS2/S4シグナル計算 → 発注案（銘柄・金額・売買）を通知
2. 日次: コア・サテライト合算のDD監視（-15%警告 / -20%戦略停止判断通知）、コア配分乖離監視（±5%）
3. 状況更新レポート（05-status-update-design.md）: 上記を snapshot API に反映
4. ペーパートレード台帳: サテライトの仮想約定を記録し、バックテストロジックとの一致を毎月検証
